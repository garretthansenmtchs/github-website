Standard 4.1 - Demonstrate Understanding of Website Planning

Wireframes are used to plan out the website. It is important to create wireframes so that web designers can create the website to the client's specifications. Wireframes give the site creator a good idea of how they want the website to look before they start creating the website.

Standard 4.2 - Demonstrate Understanding of Site Organization

Site organization is important because a website that is unnavigable will turn users away from the website. This website that you are on right now, for example, has a side menu with links to every page, and can be accessed by any page on this website. Site creators will often create a visual map to show the hierarchy of a website so that they can ensure that every pagee is accessible.

Standard 4.3 - Demonstrate Understanding of Website Accessibility Standards

Websites must be accessible for all users, but must also be aware of their audience. All websites should have some sort of backup to assist users that may be disabled in some way. For example, HTML image tags should have alt attributes, so that users can know what image is supposed to be displayed if their browser can't display it or they are blind. Websites should also account for colorblind users, and make it easy for them to see information on a page.
